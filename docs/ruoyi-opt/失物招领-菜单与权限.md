# RuoYi 后台操作指南｜失物招领 菜单与权限

目标：为“失物招领”模块配置目录/菜单/按钮权限，使管理端与门户端接口对齐可用。

## 1）新增目录与菜单
1. 系统管理 → 菜单管理 → 新增
   - 菜单类型：目录
   - 菜单名称：失物招领
   - 路由地址：/manage/lostfound
   - 显示顺序：自定
   - 图标：aim（或自选）

2. 在“失物招领”目录下新增“已发布”菜单：
   - 菜单类型：菜单
   - 菜单名称：已发布
   - 路由地址：/manage/lostfound/index
   - 组件路径：manage/lostfound/index（List.vue）
   - 权限标识：`manage:lostfound:list`

3. 新增“待审核”菜单：
   - 菜单类型：菜单
   - 菜单名称：待审核
   - 路由地址：/manage/lostfound/audit
   - 组件路径：manage/lostfound/audit（Audit.vue）
   - 权限标识：`manage:lostfound:audit:list`

4. 新增“回收站”菜单：
   - 菜单类型：菜单
   - 菜单名称：回收站
   - 路由地址：/manage/lostfound/recycle
   - 组件路径：manage/lostfound/recycle（Recycle.vue）
   - 权限标识：`manage:lostfound:recycle:list`

## 2）按钮权限（与后端一致）
- 已发布列表（List）：
  - 下架：`manage:lostfound:offline`
  - 编辑：`manage:lostfound:edit`
- 待审核（Audit）：
  - 通过：`manage:lostfound:audit:approve`
  - 驳回：`manage:lostfound:audit:reject`
- 回收站（Recycle）：
  - 恢复为待审：`manage:lostfound:restore`
  - 删除：`manage:lostfound:remove`
- 详情（通用）：
  - 查看详情：`manage:lostfound:get`
  - 新增（如需管理端创建）：`manage:lostfound:add`
 - 已发布列表（可选）：
   - 标记解决/未解决：`manage:lostfound:solve`

前端使用 v-hasPermi 与上述权限标识一致，例如：
```vue
<el-button v-hasPermi="['manage:lostfound:offline']">下架</el-button>
```

## 3）角色授权建议
- 管理员（admin/super_admin）：勾选全部。
- 其他业务角色：如需协助审核，可分配待审/下架相关按钮。

## 4）常见校验
- 403：按钮权限不匹配 → 检查标识拼写并重新登录。
- 门户端接口：无需后台按钮权限，但需保持登录态（Bearer Token）。
